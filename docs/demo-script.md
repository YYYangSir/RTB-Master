# 正式演示脚本

## 演示目标

用约 3 分钟展示从主播创建竞拍，到两个用户实时竞价、自动成交和模拟支付的完整闭环，并补充一个异常取消或自动延时场景。

## 演示前准备

在项目根目录执行：

```bash
docker compose -f infra/docker-compose.yml up -d
pnpm dev
```

打开三个页面：

| 页面 | 地址 | 用途 |
|---|---|---|
| 主播端 | `http://localhost:5173` | 创建并控制竞拍 |
| 用户端 A | `http://localhost:5174` | 第一位竞拍用户 |
| 用户端 B | `http://localhost:5174` | 第二位竞拍用户 |

## 主流程演示

### 1. 主播创建竞拍

在主播端保留默认示例或输入：

```text
商品：演示珠宝
直播间：珠宝竞拍直播间
起拍价：0 元
固定加价：100 元
封顶价：500 元
时长：120 秒
```

点击“创建竞拍”。

讲解要点：

- 主播端将商品、直播间和竞拍规则一次性创建。
- 所有金额使用整数分处理，避免浮点误差。

### 2. 分享竞拍 ID 并开拍

点击“复制竞拍 ID”，分别粘贴到用户端 A 和用户端 B。

两个用户分别：

1. 输入昵称。
2. 点击“创建演示用户”。
3. 输入竞拍 ID。
4. 点击“加入直播间”。

主播点击“开始竞拍”。

讲解要点：

- Socket.IO 将用户加入对应竞拍房间。
- 页面收到服务端快照并展示当前价格、倒计时和排行榜。

### 3. 双用户实时出价

按顺序操作：

```text
用户 A：出价 100 元
用户 B：出价 200 元
用户 A：出价 300 元
用户 B：出价 400 元
用户 A：出价 500 元
```

观察：

- 三个页面实时更新当前价和领先者。
- 用户被超越时会收到提示。
- 排行榜和参与人数同步变化。
- 达到封顶价后立即成交。

### 4. 赢家支付

用户 A 页面展示成交订单，点击“模拟支付”。

观察：

- 订单状态从 `PENDING_PAYMENT` 变为 `PAID`。
- 同一场竞拍只生成一个订单。

## 加分场景

### 自动延时

创建一场短时竞拍。在剩余时间小于等于 `10` 秒时提交合法出价：

```text
结束时间自动延长 20 秒
→ 主播端和用户端收到延时提示
→ 所有页面同步新的倒计时
```

### 异常取消

在一场运行中的竞拍点击“异常取消”：

```text
状态变为 CANCELLED
→ 用户端收到取消提示
→ 后续出价被拒绝
→ 不生成订单
```

## 自动化演示备用方案

现场操作受阻时，可直接运行：

```bash
pnpm smoke
pnpm e2e
```

`pnpm e2e` 会使用本机 Chrome 自动完成真实页面闭环。

## 结束总结

建议用一句话收尾：

> 系统不仅完成了主播开拍、用户实时竞价、成交和支付闭环，还通过 Redis Lua 保证同一场竞拍严格有序，并用自动化测试和压测结果验证了可靠性。
