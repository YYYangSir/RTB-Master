# 竞拍规则修复记录

## 问题

手动演示发现：

1. 当前领先用户可以连续给自己加价。
2. 主播端刷新页面或使用 API 预建竞拍后，缺少载入已有竞拍的入口。
3. 倒计时归零后缺少自动成交或流拍。

## 修复

- Redis Lua 增加领先者校验：当前领先用户再次出价会被拒绝。
- 主播端增加“载入已有竞拍”入口，载入后自动订阅实时房间。
- 服务端增加到期调度器，每 `500ms` 检查到期竞拍。
- 到期时通过 Redis Lua 原子结算：
  - 有领先者：状态变为 `SOLD`，生成唯一订单并广播 `auctionEnded`。
  - 无人出价：状态变为 `UNSOLD`，不生成订单并广播 `auctionEnded`。

## 竞拍时间规则

```text
每件拍品设置一个总竞拍时长
→ 最后 10 秒有人合法加价时延长 20 秒
→ 可重复触发延时
→ 达到封顶价立即成交
→ 倒计时归零时，有领先者成交，无人出价流拍
```
