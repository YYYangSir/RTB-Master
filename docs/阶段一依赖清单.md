# 阶段一必须依赖清单

> 用途：完成阶段 1 的本地环境准备，并为阶段 2 的数据库迁移、API 开发和联调提供运行基础。  
> 安全要求：禁止将真实密码、API Key、Token 或 `.env` 文件提交到 Git 仓库。

## 1. 必须安装的系统软件

以下软件必须安装在开发机器上。

| 软件 | 推荐版本 | 是否必须 | 用途 |
|---|---:|---:|---|
| Node.js | LTS，建议 `22.x` | 是 | 运行前端、后端和构建工具 |
| npm | 随 Node.js 安装 | 是 | 安装 `pnpm` |
| pnpm | `10.x` | 是 | Monorepo 依赖管理和脚本执行 |
| Docker Desktop | 当前稳定版 | 是 | 本地运行 MySQL 和 Redis |
| Git | `2.x` | 是 | 本地版本控制 |

说明：

- 当前项目不要求安装本机版 MySQL 和 Redis。
- MySQL 和 Redis 通过 Docker Compose 启动。
- MVP 不需要任何 AI API，也不需要安装 AI SDK。

## 2. 安装顺序

### 2.1 安装 Node.js LTS

安装完成后验证：

```bash
node --version
npm --version
```

### 2.2 安装 pnpm

```bash
npm install -g pnpm
pnpm --version
```

### 2.3 安装 Docker Desktop

启动 Docker Desktop 后验证：

```bash
docker --version
docker compose version
```

### 2.4 验证 Git

```bash
git --version
```

## 4. Docker 服务依赖

文件：`infra/docker-compose.yml`

| 服务 | 镜像 | 本地端口 | 用途 |
|---|---|---:|---|
| MySQL | `mysql:8.4` | `3306` | 商品、竞拍、出价和订单持久化 |
| Redis | `redis:7.4-alpine` | `6379` | 高频竞拍状态、幂等键和排行榜 |

启动：

```bash
docker compose -f infra/docker-compose.yml up -d
```

检查：

```bash
docker compose -f infra/docker-compose.yml ps
```

停止：

```bash
docker compose -f infra/docker-compose.yml down
```

## 5. 本地初始化命令

系统软件全部安装后，在项目根目录执行：

```bash
pnpm env:setup
pnpm install
docker compose -f infra/docker-compose.yml up -d
pnpm db:generate
pnpm db:migrate
pnpm dev
```

## 6. 阶段一完成检查

在项目根目录执行：

```bash
bash scripts/check-environment.sh
```

预期结果：

```text
[ok] node
[ok] pnpm
[ok] docker
[ok] git
Environment check passed.
```

随后检查容器：

```bash
docker compose -f infra/docker-compose.yml ps
```

阶段 1 依赖准备完成的标准：

- `node`、`npm`、`pnpm`、`docker`、`docker compose`、`git` 均可执行。
- MySQL 和 Redis 容器健康运行。
- `pnpm install` 成功完成。
- `.env` 已创建但没有被 Git 跟踪。
- `pnpm db:generate` 可以执行。
- 项目中不存在真实 API Key 或 Token。

## 7. 当前不需要安装的依赖

以下内容不是阶段 1 必需项：

- AI 模型 SDK。
- 火山方舟、OpenAI 或其他模型 API 客户端。
- 本机版 MySQL。
- 本机版 Redis。
- Kubernetes。
- 云数据库。
- 对象存储 SDK。
- 真实支付 SDK。
- 直播推流 SDK。
