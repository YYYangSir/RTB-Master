## 2. 核心功能

- 主播创建商品、直播间和竞拍，并配置价格规则。
- 两个用户进入同一直播间，通过 Socket.IO 实时竞价。
- Redis Lua 原子处理幂等、最低价、自动延时和封顶成交。
- 倒计时结束自动结算：有领先者成交，无人出价流拍。
- 实时排行榜、被超越提醒、异常取消和断线恢复。
- 唯一订单生成和模拟支付。
- API smoke、浏览器 E2E 和单实例压测。

## 5. 工程亮点

### 亮点一：Redis Lua 原子竞价

同一场竞拍的出价在 Redis 中严格顺序处理，避免价格覆盖、重复请求和事务冲突。

### 亮点二：完整可验证闭环

不仅可以手动演示，还提供规则测试、API smoke、真实 Chrome E2E 和性能压测。

### 亮点三：双端实时体验

主播端和用户端同步展示价格、倒计时、排行榜、被超越提示、延时和成交结果。

## 6. 工程难点

| 难点 | 解决方案 |
|---|---|
| 并发出价覆盖 | Redis Lua 原子更新热点状态 |
| 重复点击和网络重试 | `requestId` 幂等键和数据库唯一索引 |
| 重复订单 | `orders.auctionId UNIQUE` 和数据库 `upsert` |
| 页面断线恢复 | Socket.IO 重连后重新加入房间并获取快照 |
| 排行榜高频聚合 | Redis ZSET 保存用户最高出价 |

## 7. AI 能力说明

当前系统运行时不调用外部 AI API，不需要上传任何模型 API Key。

后续可选增强：

- 商家端推荐起拍价、加价幅度和预计成交区间。
- 买家端提供保守和进取两档出价建议。
- AI 仅用于解释、文案和复盘，不进入实时竞价关键链路。
