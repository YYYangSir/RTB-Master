## 详细问题

### BLOCKER-001：MySQL 未运行导致 API 无法启动

严重程度：阻断级

现象：

```text
PrismaClientInitializationError: Can't reach database server at `127.0.0.1:3306`
```

影响：

- 后端 API 无法启动。
- 商品创建、竞拍创建、出价、订单、WebSocket 连接等所有业务测试均无法执行。
- `TC-RULE-001` 及后续测试全部依赖该前置条件，因此本轮测试必须暂停。

判断：

- 这不是业务逻辑缺陷，而是测试环境未就绪。
- 需要先启动项目依赖的 MySQL 和 Redis 容器。
- 虽然端口 `3306` 有监听，但 Prisma 仍无法完成数据库连接，说明需要检查 Docker Desktop 中 `auction-mysql` 容器是否真正 healthy，以及数据库账号、端口映射是否正常。
- 用户提供的容器状态显示 `auction-mysql` 仍处于 `health: starting`，不是 healthy。即使端口有监听，MySQL 也可能尚未完成初始化，或者健康检查失败。

### BLOCKER-002：当前执行环境无权限访问 Docker API

严重程度：阻断级

现象：

```text
permission denied while trying to connect to the docker API at unix:///Users/yzy/.docker/run/docker.sock
```

影响：

- 无法通过命令确认 `auction-mysql`、`auction-redis` 是否健康。
- 无法自动启动或重启容器。
- 无法继续可靠执行测试。
